## Authentication Methods

MemeWarpAI supports two primary authentication methods:

### Email/Password Authentication

Traditional email and password authentication is implemented using Firebase Authentication. This method is suitable for users who prefer a conventional login experience.

### Web3 Wallet Authentication

For Web3 users, MemeWarpAI supports authentication using cryptocurrency wallets like MetaMask, WalletConnect, and other Ethereum-compatible wallets. This method uses cryptographic signatures to verify wallet ownership without requiring a password.

## Registration Flow

### Email/Password Registration

1. User provides email, password, and optional username
2. System validates email format and password strength
3. Firebase Authentication creates a new user account
4. A user document is created in Firestore with initial settings
5. User receives a welcome email with verification link
6. User is automatically logged in after successful registration

### Wallet Registration

1. User connects their wallet using RainbowKit
2. System generates a unique message for the user to sign
3. User signs the message using their wallet
4. System verifies the signature to confirm wallet ownership
5. A user document is created in Firestore with the wallet address as the user ID
6. User is automatically logged in after successful verification

## Login Flow

### Email/Password Login

1. User enters email and password
2. System authenticates with Firebase Authentication
3. Upon successful authentication, user data is loaded from Firestore
4. User is redirected to the appropriate page based on their intended action

### Wallet Login

1. User connects their wallet using RainbowKit
2. System checks if the wallet address exists in the database
3. If the wallet is recognized, the user is automatically logged in
4. If the wallet is new, the user is prompted to complete registration
5. User data is loaded from Firestore after successful authentication

## Token Management

MemeWarpAI uses JWT (JSON Web Tokens) for authentication and session management.

### Token Generation

- Tokens are generated by Firebase Authentication upon successful login
- Tokens contain the user's ID and authentication claims
- Tokens are signed with a private key to ensure integrity

### Token Storage

- Tokens are stored in the browser's localStorage for persistence
- The AuthContext in the application manages token state
- Token expiration is handled automatically by Firebase

### Token Refresh

- Tokens are automatically refreshed before expiration
- The refresh process is handled by Firebase Authentication
- If a refresh fails, the user is prompted to log in again

## Security Considerations

### Password Security

- Passwords are never stored in plaintext
- Firebase Authentication handles password hashing and security
- Password strength requirements are enforced during registration
- Password reset functionality is available via email

### Wallet Security

- Private keys never leave the user's wallet
- Only message signatures are transmitted, not private keys
- Each signature request includes a unique nonce to prevent replay attacks
- Clear messaging informs users what they are signing

### General Security

- All API requests are authenticated using JWT tokens
- HTTPS is enforced for all communications
- CORS policies restrict API access to authorized domains
- Rate limiting is implemented to prevent brute force attacks

## Integration with Web3 Wallets

MemeWarpAI integrates with Web3 wallets using the following components:

### RainbowKit

RainbowKit provides a user-friendly interface for connecting various wallets:

```javascript
import { ConnectButton } from '@rainbow-me/rainbowkit';

// In your component
<ConnectButton />
```

### Wagmi

Wagmi is used for interacting with Ethereum and managing wallet connections:

```javascript
import { useWalletClient } from 'wagmi';

// In your component
const { data: walletClient } = useWalletClient();
```

### Authentication Flow

The wallet authentication flow is implemented as follows:

```javascript
// 1. Get the wallet client
const { data: walletClient } = useWalletClient();

// 2. Sign a message
const message = `Sign this message to authenticate with MemeWarpAI: ${nonce}`;
const signature = await walletClient.signMessage({ message });

// 3. Verify the signature on the server
const response = await fetch('/api/auth/wallet', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    walletAddress: walletClient.account.address,
    signature,
    message
  })
});

// 4. Handle the authentication result
const result = await response.json();
if (result.success) {
  // Store the token and user data
  localStorage.setItem('token', result.token);
  setCurrentUser(result.user);
}
```
